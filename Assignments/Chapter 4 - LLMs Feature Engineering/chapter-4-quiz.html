Module 4 Quiz: LLMs for Feature Engineering and Data Extraction

Instructions

Time: 30-45 minutes Format: In-class, by-hand (no computer)

Answer all questions. Write code by hand as neatly as possible. Partial credit will be given for correct reasoning even if syntax isn’t perfect.

Section A: Conceptual Questions

Question 1 (3 points)

You have three text extraction tasks:

Task A: Extract email addresses from customer support tickets (format: name@company.com)

Task B: Extract sentiment (positive/negative/neutral) from product reviews with complex, nuanced language

Task C: Extract dates from legal documents (format: MM/DD/YYYY or Month DD, YYYY)

a) For which task(s) would using an LLM API be most appropriate? For which would regex or traditional methods be better?

b) Explain your reasoning for each choice.

Question 2 (3 points)

Compare these two prompts for extracting product categories from reviews:

Prompt A: “What category is this product?”

Prompt B: “Read this product review and extract the product category. Return your answer as JSON with a single field ‘category’. Valid categories are: Electronics, Clothing, Home & Garden, Books, Toys. Review: {review_text}”

a) Which prompt will produce more reliable, consistent extractions?

b) Explain two specific improvements Prompt B makes over Prompt A.

Question 3 (4 points)

You need to extract sentiment from 1,000 customer reviews.

Given:

  • GPT-4: $0.03 per 1,000 input tokens, $0.06 per 1,000 output tokens
  • GPT-3.5-turbo: $0.001 per 1,000 input tokens, $0.002 per 1,000 output tokens
  • Average review: 150 tokens
  • Average response: 20 tokens

a) Calculate the total cost to process all 1,000 reviews using GPT-4.

b) Calculate the total cost using GPT-3.5-turbo.

c) If GPT-4 achieves 95% accuracy and GPT-3.5-turbo achieves 88% accuracy, which would you choose and why?

Question 4 (3 points)

You receive these three JSON responses from an LLM extraction task:

Response 1: {"sentiment": "positive", "rating": 4}

Response 2: {"sentiment": "positive" "rating": 4}

Response 3: {"sentiment": "positive", "rating": "four"}

a) Which response(s) are valid JSON? Which are invalid?

b) For the problematic responses, explain what’s wrong and how you would handle them in your code.

Question 5 (3 points)

You’re using an LLM API to extract information from text. Explain the effect of each parameter:

a) What happens when you set temperature=0 vs temperature=1.0? When would you use each setting for extraction tasks?

b) What does max_tokens control? Why might you set it to a low value like 50 for extraction tasks?

Question 6 (3 points)

You extract sentiment from 100 product reviews and manually check the results:

  • 92 extractions match human judgment
  • 5 extractions are close but slightly off
  • 3 extractions are completely wrong

a) Is this extraction quality acceptable for use in a machine learning pipeline? Explain your reasoning.

b) Describe two approaches you could take to improve the extraction quality.

Question 7 (3 points)

You’re building a spam classifier. You have a dataset with email text and labels (spam/not spam). You want to use an LLM to extract additional features like “urgency level” and “contains financial terms.”

a) At what point in your workflow should you extract these LLM features to avoid data leakage?

b) After extracting the features, how would you add them to your existing DataFrame before training a model?

Question 8 (2 points)

a) What is few-shot prompting?

b) When would you use few-shot prompting instead of zero-shot prompting for extraction tasks?

Section B: Code Writing

import pandas as pd

Question 9 (3 points)

Write a prompt (as a Python string) to extract sentiment from a product review. The prompt should:

  1. Ask the LLM to analyze the review text
  2. Request the response in JSON format with a field called “sentiment”
  3. Specify that sentiment should be one of: “positive”, “negative”, or “neutral”

Store the prompt in a variable called prompt.

Question 10 (3 points)

You receive this JSON string response from an LLM:

response_text = '{"sentiment": "positive", "confidence": 0.95}'

Write code to:

  1. Parse the JSON string into a Python dictionary
  2. Extract the “sentiment” value
  3. Store it in a variable called sentiment

Question 11 (3 points)

You have a list of extraction results from processing multiple reviews:

results = [
    {"sentiment": "positive", "rating": 5},
    {"sentiment": "negative", "rating": 2},
    {"sentiment": "neutral", "rating": 3}
]

Write code to convert this list into a pandas DataFrame called df with columns “sentiment” and “rating”.

Question 12 (4 points)

You have two DataFrames:

reviews_df  # columns: review_id, review_text
extracted_df  # columns: review_id, sentiment, urgency

Write code to merge these DataFrames so that the sentiment and urgency features are added to the original reviews DataFrame. Use review_id as the key for merging. Store the result in combined_df.

Question 13 (4 points)

Write code to make an API call to extract sentiment from a review. Your code should:

  1. Create a message with the system role: “You extract sentiment from reviews”
  2. Use the user role with: “Extract sentiment from this review: {review_text}. Return JSON.”
  3. Call client.chat.completions.create() with model=“gpt-3.5-turbo”
  4. Set temperature=0 and max_tokens=50

Store the response in a variable called response.

(Note: You can use simplified syntax; exact parameter names don’t need to be perfect)

Question 14 (4 points)

Write a function called calculate_extraction_cost that:

  • Takes three parameters: num_texts, tokens_per_text, and price_per_1k_tokens
  • Calculates the total tokens as num_texts * tokens_per_text
  • Calculates the cost as (total_tokens / 1000) * price_per_1k_tokens
  • Returns the total cost

Question 15 (4 points)

You have a DataFrame df with columns: review_text, llm_sentiment, and label (the true sentiment).

Write code to:

  1. Split the data into train and test sets (80/20 split, random_state=42)
  2. Use only the llm_sentiment feature (needs to be 2D: use double brackets)
  3. Train a Logistic Regression model on the training data
  4. Calculate and print the accuracy on the test set
