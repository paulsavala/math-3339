Quiz 4b: JSON Parsing, Quality Control, and ML Integration

Instructions

Time: 30-45 minutes Format: In-class, by-hand (no computer)

Answer all questions. Write code by hand as neatly as possible. Partial credit will be given for correct reasoning even if syntax isn’t perfect.

Section A: Conceptual Questions

Question 1 (3 points)

You receive these three JSON responses from an LLM extraction task:

Response 1: {"sentiment": "positive", "rating": 4}

Response 2: {"sentiment": "positive" "rating": 4}

Response 3: {"sentiment": "positive", "rating": "four"}

a) Which response(s) are valid JSON? Which are invalid?

b) For the problematic responses, explain what’s wrong and how you would handle them in your code.

Question 2 (3 points)

You extract sentiment from 100 product reviews and manually check the results:

  • 92 extractions match human judgment
  • 5 extractions are close but slightly off
  • 3 extractions are completely wrong

a) Is this extraction quality acceptable for use in a machine learning pipeline? Explain your reasoning.

b) Describe two approaches you could take to improve the extraction quality.

Question 3 (3 points)

You’re building a spam classifier. You have a dataset with email text and labels (spam/not spam). You want to use an LLM to extract additional features like “urgency level” and “contains financial terms.”

a) At what point in your workflow should you extract these LLM features to avoid data leakage?

b) After extracting the features, how would you add them to your existing DataFrame before training a model?

Question 4 (2 points)

a) What is few-shot prompting?

b) When would you use few-shot prompting instead of zero-shot prompting for extraction tasks?

Question 5 (3 points)

When integrating LLM-extracted features into a machine learning pipeline:

a) Why is it important to extract features from the test set AFTER splitting the data?

b) What problem could arise if you extract features before splitting?

c) How do you ensure the LLM extraction process is the same for training and test data?

Section B: Code Writing

For questions 6-11, assume you have already imported:

import pandas as pd
import numpy as np
import json
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

Question 6 (3 points)

You receive this JSON string response from an LLM:

response_text = '{"sentiment": "positive", "confidence": 0.95}'

Write code to:

  1. Parse the JSON string into a Python dictionary
  2. Extract the “sentiment” value
  3. Store it in a variable called sentiment

Question 7 (3 points)

You have a list of extraction results from processing multiple reviews:

results = [
    {"sentiment": "positive", "rating": 5},
    {"sentiment": "negative", "rating": 2},
    {"sentiment": "neutral", "rating": 3}
]

Write code to convert this list into a pandas DataFrame called df with columns “sentiment” and “rating”.

Question 8 (4 points)

You have two DataFrames:

reviews_df  # columns: review_id, review_text
extracted_df  # columns: review_id, sentiment, urgency

Write code to merge these DataFrames so that the sentiment and urgency features are added to the original reviews DataFrame. Use review_id as the key for merging. Store the result in combined_df.

Question 9 (4 points)

Write code to handle a JSON parsing error gracefully:

  1. Try to parse a JSON string stored in variable response_text
  2. If successful, extract the “sentiment” field
  3. If parsing fails, set sentiment to “unknown”
  4. Store the final sentiment value in variable sentiment

Use a try-except block.

Question 10 (3 points)

You have a DataFrame df with a column llm_response containing JSON strings like '{"category": "Electronics"}'.

Write code to:

  1. Parse each JSON string in the column
  2. Extract the “category” value from each
  3. Create a new column called category with the extracted values

You can use a loop or .apply().

Question 11 (4 points)

You have a DataFrame df with columns: review_text, llm_sentiment, and label (the true sentiment).

Write code to:

  1. Split the data into train and test sets (80/20 split, random_state=42)
  2. Use only the llm_sentiment feature (needs to be 2D: use double brackets)
  3. Train a Logistic Regression model on the training data
  4. Calculate and print the accuracy on the test set
